# 치과 AI 아이디어 로그북 - 개발 로드맵

## Phase 1: 프로젝트 초기 설정 (1-2일)

### 작업 내용
- Next.js 15 프로젝트 구조 설정 (TypeScript)
- Notion API 연동 환경 구축 (`NOTION_API_KEY`, `NOTION_DATABASE_ID`)
- 기본 레이아웃 구조 생성 (`app/layout.tsx`)
- Tailwind CSS, shadcn/ui 초기 설정
- 프로젝트 폴더 구조 확립 (`components/`, `lib/`, `types/`)

### 이 순서인 이유
견고한 기반 없이는 기능 개발이 어려움. 환경 설정과 프로젝트 구조가 먼저 갖춰져야 이후 개발이 원활하게 진행됨.

### 예상 소요 시간
1-2일

### 완료 기준
- [ ] Next.js 15 프로젝트가 정상 실행됨
- [ ] Notion API 연결 테스트 성공 (데이터 조회 확인)
- [ ] Tailwind CSS, shadcn/ui가 정상 동작함
- [ ] 기본 레이아웃이 브라우저에 렌더링됨

---

## Phase 2: 공통 모듈 개발 (2-3일)

### 작업 내용
- Notion API 공통 함수 (`lib/notion.ts`)
  - `fetchIdeas()`: 아이디어 목록 조회
  - `fetchIdeaById()`: 단일 아이디어 조회
- 공통 컴포넌트 개발
  - `Header`: 사이트 헤더
  - `Footer`: 사이트 푸터
  - `TagBadge`: 태그 배지 컴포넌트
- 공통 타입 정의 (`types/idea.ts`)
  - `Idea`: 아이디어 타입
  - `Target`, `Category`: 태그 타입
  - `BusinessPotential`, `TechnicalDifficulty`: 평가 타입

### 이 순서인 이유
모든 기능에서 재사용되는 코드를 먼저 만들어야 중복 방지. 타입 정의가 선행되어야 타입 안전성 확보 가능.

### 예상 소요 시간
2-3일

### 완료 기준
- [ ] Notion API에서 아이디어 목록을 정상적으로 가져옴
- [ ] 공통 컴포넌트가 독립적으로 렌더링됨
- [ ] 모든 타입이 정의되고 any 타입 없이 사용 가능

---

## Phase 3: 핵심 기능 개발 (3-4일)

### 작업 내용
- 아이디어 목록 페이지 (`app/page.tsx`)
  - `IdeaCard` 컴포넌트 개발
  - 최신순 정렬 표시
  - 제목, 타겟 태그, 사업성/난이도 배지 표시
- 아이디어 상세 페이지 (`app/ideas/[id]/page.tsx`)
  - `IdeaDetail` 컴포넌트 개발
  - 문제 정의, 실제 사례, AI 접근 방식 섹션
  - 평가 정보 (사업성, 난이도, 대상, 카테고리) 표시

### 이 순서인 이유
아이디어 조회가 서비스의 가장 기본이 되는 핵심 기능. MVP의 필수 요소이므로 우선 개발.

### 예상 소요 시간
3-4일

### 완료 기준
- [ ] 메인 페이지에서 아이디어 카드 목록이 최신순으로 표시됨
- [ ] 카드 클릭 시 상세 페이지로 이동
- [ ] 상세 페이지에서 모든 필드가 구조화되어 표시됨
- [ ] 반응형 레이아웃 적용 완료

---

## Phase 4: 추가 기능 개발 (2-3일)

### 작업 내용
- 태그 기반 필터링 기능
  - 대상별 필터 (의사/환자/스태프/기공사/학생)
  - 영역별 필터 (진단/상담/운영/교육/행정)
- 검색 기능
  - 제목 기반 검색
- SEO 최적화
  - 메타데이터 설정
  - Open Graph 태그 추가

### 이 순서인 이유
핵심 기능이 완성된 후 사용자 경험을 향상시키는 부가 기능 추가. MVP 범위에서는 선택 사항이지만 실용성을 높임.

### 예상 소요 시간
2-3일

### 완료 기준
- [ ] 태그 클릭 시 해당 태그의 아이디어만 필터링됨
- [ ] 검색어 입력 시 관련 아이디어가 검색됨
- [ ] 페이지별 적절한 메타데이터가 설정됨

1-2일
